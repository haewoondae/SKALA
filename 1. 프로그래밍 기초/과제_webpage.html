1. 인터넷과 웹

기초 개념

📚 개념

인터넷과 웹의 차이점

  • 인터넷 - 전 세계를 연결하는 물리적 네트워크의 집합
  • 웹(WWW) - 인터넷에서 사용할 수 있는 서비스 중 하나
  • 프로토콜 - HTTP/HTTPS, TCP/IP, DNS 등 통신 규약
  • 클라이언트-서버 모델 - 요청과 응답으로 이루어진 웹의 기본 구조

⚙️ 웹 기술 요소

  • HTTP/HTTPS - 웹 페이지 전송 프로토콜
  • DNS - 도메인 이름을 IP 주소로 변환하는 시스템
  • URL - 웹 리소스의 위치를 지정하는 주소 체계

2. HTML 개요

기초 학습

📚 개념

HTML (Hypertext Markup Language)

웹 페이지의 구조와 콘텐츠를 정의하는 마크업 언어

  • 마크업 언어 - 태그를 사용해 문서 구조를 정의하는 언어
  • 플랫폼 독립성 - 모든 브라우저와 운영체제에서 동작
  • 시맨틱 구조 - 의미있는 태그로 문서의 논리적 구조 표현

🏷️ 태그

  • <!DOCTYPE html> - HTML5 문서임을 선언하는 문서형 정의
  • <html> - HTML 문서의 루트 요소, 모든 요소를 포함
  • <head> - 메타데이터 영역, 문서 정보와 스타일, 스크립트 포함
  • <body> - 본문 영역, 사용자에게 실제로 표시될 콘텐츠
  • <meta> - 문서의 메타데이터 정의 (문자셋, 뷰포트 등)
  • <title> - 브라우저 탭에 표시되는 문서 제목

3. 개발환경 구성

환경 설정

📚 개념

HTML 개발을 위한 필수 도구들

  • 코드 에디터 - HTML, CSS, JavaScript 코드를 작성하는 도구

4. 텍스트 및 콘텐츠 태그

기본 태그

📚 개념

웹 페이지의 텍스트 내용을 구조화하고 의미를 부여하는 태그들

  • 제목 태그 - 문서의 계층적 구조를 나타내는 h1~h6 태그
  • 문단 태그 - 텍스트를 논리적 단위로 구분하는 p 태그
  • 하이퍼링크 - 웹의 핵심인 링크 기능을 제공하는 a 태그
  • 텍스트 서식 - 텍스트의 의미와 강조를 표현하는 다양한 태그들

🏷️ 태그

  • <h1>~<h6> - 제목 태그, h1이 가장 크고 중요하며 h6이 가장 작음, SEO에 중요한 역할
  • <p> - 문단 태그, 블록 요소로 자동 줄바꿈과 여백 제공
  • <br> - 줄바꿈 태그, 강제 줄바꿈을 위한 빈 태그
  • <hr> - 수평선 태그, 콘텐츠 구분을 위한 선 삽입
  • <a> - 앵커(링크) 태그, href 속성으로 링크 대상 지정
  • <strong> - 중요성 강조 태그, 의미적으로 중요한 내용 표시
  • <b> - 굵은 글씨 태그, 단순한 시각적 효과
  • <i> - 기울임 글씨 태그, 단순한 시각적 효과
  • <mark> - 하이라이트 태그, 형광펜 효과로 내용 강조
  • <span> - 인라인 구획 태그, 스타일링 목적으로 사용
  • <div> - 블록 구획 태그, 레이아웃 구성과 그룹화에 사용
  • contenteditable - 편집 가능 속성, 사용자가 직접 내용을 수정할 수 있게 함

5. 목록과 테이블 태그

구조 태그

📚 개념

정보를 구조화하여 표시하는 태그들

  • 순서 있는 목록 - 항목들을 번호나 문자 순서대로 나열
  • 순서 없는 목록 - 항목들을 불릿 포인트로 나열
  • 정의 목록 - 용어와 설명의 쌍으로 구성된 목록
  • 테이블 - 행과 열의 격자 형태로 데이터를 체계적으로 표시

🏷️ 태그

  • <ol> - 순서 있는 목록, type 속성으로 번호 스타일 변경 가능 (1, A, a, I, i)
  • <ul> - 순서 없는 목록, CSS로 불릿 스타일 변경 가능
  • <li> - 목록 항목, ol이나 ul 내부에서 각 항목을 나타냄
  • <dl> - 정의 목록, 용어와 설명의 쌍으로 구성
  • <dt> - 정의 용어, 설명할 용어나 제목
  • <dd> - 정의 설명, dt에 대한 상세 설명
  • <table> - 테이블 컨테이너, 표 전체를 감싸는 최상위 태그
  • <thead> - 테이블 헤더 그룹, 제목 행들을 묶음
  • <tbody> - 테이블 본문 그룹, 데이터 행들을 묶음
  • <tr> - 테이블 행, 표의 가로줄
  • <th> - 헤더 셀, 굵은 글씨로 표시되는 제목 셀
  • <td> - 데이터 셀, 일반적인 데이터가 들어가는 셀

6. 폼(사용자 입력) 태그

입력 태그

📚 개념

사용자로부터 데이터를 입력받는 태그들

  • 폼 컨테이너 - 사용자 입력 요소들을 묶어서 서버로 전송
  • 입력 필드 - 다양한 타입의 데이터를 입력받는 요소
  • 선택 메뉴 - 미리 정의된 옵션 중에서 선택할 수 있는 드롭다운
  • 버튼 - 사용자 액션을 트리거하는 클릭 가능한 요소

🏷️ 태그

  • <form> - 폼 컨테이너, action과 method 속성으로 데이터 전송 방식 설정
  • <input> - 입력 필드, type 속성으로 다양한 입력 타입 지정 (text, email, password 등)
  • <select> - 드롭다운 선택 메뉴, multiple 속성으로 다중 선택 가능
  • <option> - 선택 항목, select 내부의 각 선택 가능한 항목
  • <button> - 버튼 요소, type 속성으로 submit, button, reset 타입 설정
  • <label> - 라벨 요소, for 속성으로 특정 input과 연결하여 접근성 향상
  • <textarea> - 여러 줄 텍스트 입력 영역, rows와 cols 속성으로 크기 조절

7. 미디어 태그

미디어

📚 개념

웹 페이지에 이미지, 오디오, 비디오 등 미디어 콘텐츠를 삽입하는 태그들

  • 이미지 - 정적 시각 콘텐츠를 표시하는 기본 미디어
  • 오디오 - 음성, 음악 등 소리 콘텐츠 재생
  • 비디오 - 동영상 콘텐츠 재생과 컨트롤
  • 반응형 미디어 - 화면 크기와 해상도에 따른 적응형 미디어

🏷️ 태그

  • <img> - 이미지 삽입, src(소스), alt(대체텍스트), width, height 속성 사용
  • <audio> - 오디오 재생, controls, autoplay, loop 속성으로 재생 옵션 설정
  • <video> - 비디오 재생, controls, poster(썸네일) 속성 등 사용
  • <source> - 미디어 소스 지정, 여러 형식 파일을 브라우저 호환성에 따라 선택
  • <picture> - 반응형 이미지 컨테이너, 미디어 쿼리와 함께 사용
  • <figure> - 독립적인 콘텐츠 블록, 이미지와 캡션을 함께 묶음
  • <figcaption> - figure의 캡션, 미디어에 대한 설명 텍스트

8. 시맨틱 태그

고급 태그

📚 개념

의미가 있는 태그로, 문서의 구조와 콘텐츠의 의미를 명확하게 정의

  • SEO 최적화 - 검색 엔진이 콘텐츠를 더 잘 이해할 수 있도록 도움
  • 접근성 향상 - 스크린 리더 등 보조 기술의 문서 해석 능력 향상
  • 코드 가독성 - 개발자가 HTML 구조를 더 쉽게 이해할 수 있음
  • 유지보수 용이성 - 명확한 의미 구조로 코드 수정과 관리가 용이

🏷️ 태그

  • <header> - 헤더 영역, 문서나 섹션의 머리글, 제목과 내비게이션 포함
  • <nav> - 네비게이션 영역, 메뉴와 링크 모음
  • <main> - 주요 콘텐츠 영역, 문서의 핵심 내용 (페이지당 하나만 사용)
  • <section> - 섹션 구분, 주제별로 콘텐츠를 그룹화
  • <article> - 독립적 콘텐츠, 뉴스 기사, 블로그 포스트 등 완전한 내용
  • <aside> - 보조 콘텐츠, 사이드바, 광고, 관련 링크 등
  • <footer> - 푸터 영역, 저작권 정보, 연락처, 추가 정보
  • <time> - 시간 정보, datetime 속성으로 기계 판독 가능한 시간 표시
  • <address> - 연락처 정보, 문서 작성자나 소유자의 연락처

        function formatText(command, value) {
            document.execCommand(command, false, value);
        }

            const editor = document.getElementById('editor');
            const output = document.getElementById('htmlOutput');
            output.textContent = editor.innerHTML;
